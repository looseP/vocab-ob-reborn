## 第一部分：算法层 —— 间隔重复算法演进

### 1.1 演进时间线

```
1985  SM-0    纸笔时代，固定间隔表
1987  SM-2    Anki 至今的基础，公式简单
1989  SM-4    引入"间隔矩阵"，可自适应
1995  SM-8    引入"遗忘曲线"反馈，难易度自适应
2011  SM-15   SuperMemo 商业版，多变量优化
2016  SM-17   "稳定性矩阵"，三变量模型
2019  SM-18   优化矩阵更新，更细粒度
2022  FSRS    开源，深度学习优化，DSR 模型
2024  FSRS v5/v6  19 参数，Anki/RemNote/MnemOS 采用
```

### 1.2 SM-0（1985）—— 一切的起点

Piotr Woźniak 在大学时用纸笔做的实验。核心思想极其简单：**固定的间隔表**。

```
重复次数    间隔
1           1 天
2           2 天
3           4 天
4           8 天
5           16 天
...         ×2
```

**关键洞察**：间隔翻倍比每天复习记得更牢。这就是"间隔效应"的最早工程化。
**局限**：完全不考虑卡片难度差异，所有卡一视同仁。

### 1.3 SM-2（1987）—— Anki 的根基

引入**易度因子（Easiness Factor, EF）**，每张卡有自己的难度系数。

**核心公式**：

```
EF' = EF + (0.1 - (5-q)*(0.08+(5-q)*0.02))

其中 q = 评级（0-5），SM-2 用 6 级：
  0 = 完全不记得
  3 = 勉强想起，有困难
  5 = 完美想起

间隔计算：
  n=1: I(1) = 1
  n=2: I(2) = 6
  n>2: I(n) = I(n-1) × EF
```

**评级对 EF 的影响**：
- q=5（完美）→ EF 增大 0.1（更易，间隔涨更快）
- q=4 → EF 不变
- q=3 → EF 减小 0.14
- q<3 → EF 大幅减小，且**重置间隔**（回到 n=1）

**为什么 Anki 至今用 SM-2**：
- 公式简单，可手算验证
- 1987 年的算法，40 年验证
- Anki 在其上加了：hard interval 倍率、easy interval 倍率、模糊匹配、deck 级配置
- **够用**。对大多数用户，SM-2 的准确度已足够

**SM-2 的根本缺陷**：
- EF 只在 q<3 时重置，q=3（勉强）会持续累积"假性稳定"——你以为记住了，其实临界
- EF 的更新是**线性启发式**，不是从数据学的
- 没有显式建模"记忆强度"和"可检索性"——只有一个 EF 当万能钥匙

### 1.4 SM-4 / SM-8 / SM-15 —— 自适应的探索期

**SM-4（1989）**：引入**间隔矩阵（OF Matrix）**。不再用公式算间隔，而是查一个 20×20 的矩阵（行=重复次数，列=EF 等级），矩阵值 = 推荐间隔。每次复习后根据实际表现**微调矩阵**。

**SM-8（1995）**：加入**遗忘曲线反馈**。要求用户在复习时报告"我花了几秒想起来"，用反应时辅助判断记忆强度。难易度（A-Factor）从卡片首次复习时的表现推断。

**SM-15（2011）**：商业 SuperMemo 的成熟版。多变量优化，引入"遗忘指数"作为目标函数——用户设定"我允许 10% 遗忘率"，系统调整间隔让实际遗忘率逼近目标。

**这一系列的意义**：从"公式驱动"走向"数据驱动"。但矩阵/启发式仍是**人为设计**，不是从数据学的。

### 1.5 SM-17 / SM-18 —— 三变量模型

**SM-17（2016）** 的核心突破：显式建模三个独立变量：

| 变量 | 含义 | 类比 |
|------|------|------|
| **Difficulty (D)** | 卡片难度（0-1） | 这张卡本身有多难记 |
| **Stability (S)** | 记忆稳定性（天） | 多久不复习会忘到临界 |
| **Retrievability (R)** | 当前可检索性（0-1） | 此刻能想起的概率 |

三者关系（**这是所有现代 SRS 的理论基石**）：

```
R = exp(-t / S)

t = 距上次复习的天数
S = 记忆稳定性（越大，R 衰减越慢）
```

复习后 S 的更新（简化）：

```
S' = S × (1 + f(D, R, rating))

f 是关于难度、当前可检索性、评级的函数
rating 好 + R 低（差点忘了但想起来了）→ S 增长最大（desirable difficulty 效应）
```

**SM-17 vs SM-2 的根本区别**：SM-2 的 EF 把"难度"和"稳定性"混为一谈；SM-17 把它们分开，因此能建模"难度高但已稳定"和"难度低但未稳定"的不同情况。

**SM-18（2019）**：优化矩阵更新算法，更细粒度的稳定性追踪。仍是商业闭源。

### 1.6 FSRS —— 开源的突破

**FSRS（Free Spaced Repetition Scheduler）**，2022 年由 Jarrett Ye 开源。**首次把深度学习用于 SRS 参数优化**。

#### DSR 模型

FSRS 沿用 SM-17 的三变量框架，但用**神经网络学参数**：

**初始稳定性（首次复习）**：
```
S0 = w[rating - 1]    // w[0..3] 是学出来的 4 个参数，对应 again/hard/good/easy 的初始稳定性
```

**记忆稳定性更新**（核心公式，简化版）：
```
// 答对的情况（rating ≥ 2）
S' = S × (e^w × (11 - D) × S^(-w) × (e^(w×(1-R)) - 1) × hard_factor + 1)

// 答错的情况（rating = 1，即 again）
S' = w × D^(-w) × ((S+1)^w - 1)

其中：
D = 难度（0-1，初始 w[4]+rating相关，每次复习动态更新）
R = 当前可检索性 = exp(-t/S)
w = 19 个可学习参数（FSRS v5）
```

**难度更新**：
```
D' = w × (D - w×(rating-3))    // rating=3(good) 不改难度，>3 降难度，<3 升难度
D' = clamp(D', 1, 10)
```

#### 为什么 FSRS 优于 SM-2

| 维度 | SM-2 | FSRS |
|------|------|------|
| 参数来源 | 人为设计的启发式公式 | 从用户真实复习数据学的（梯度下降） |
| 变量数 | 1（EF） | 3（D/S/R 显式分离） |
| 个性化 | 每卡有 EF，但更新规则全局固定 | 每用户/每 wordbook 可有独立 weights |
| 可检索性预测 | 不显式预测 | 显式预测 R = exp(-t/S)，可校准 |
| 目标函数 | 无（间隔是副作用） | 显式优化 logLoss（预测 R vs 实际回忆） |
| 数据驱动程度 | 公式固定，数据只调 EF | 19 参数全从数据学 |

**一句话**：SM-2 是"专家凭经验设计公式"，FSRS 是"让数据自己说话"。在 Anki 的基准测试里，FSRS 的预测准确度（logLoss）比 SM-2 低 15-30%。

#### FSRS 的 desired_retention

FSRS 引入了一个 SM 系列没有的概念：**目标留存率**（desired_retention）。

```
系统调整间隔，使 R 在到期时恰好 ≈ desired_retention
```

你的项目用 0.9（90%），含义是"每张卡到期时，系统让你有 90% 概率能想起"。

- 调高（0.95）→ 间隔变短，复习更频繁，但更少遗忘
- 调低（0.85）→ 间隔变长，复习更少，但遗忘更多

**这是一个用户可调的旋钮**——备考冲刺期调高，平时维持调低。SM-2 没有这个概念。

#### 你的项目与 FSRS

你已经在用 `ts-fsrs`（FSRS v5/v6）+ native binding 优化器：
- wordbook 级 weights（每个 wordbook 可独立训练 19 参数）
- desired_retention 可配（0.7-0.99，默认 0.9）
- weights 训练门槛 500 条复习记录
- adapter 解耦（v2 的 ReviewService 不直接依赖 ts-fsrs）

**你已经在算法层用了当前最优方案**。后续创新不在于换算法，而在于**在 FSRS 之上叠加新的信号维度**。

### 1.7 Half-Life Regression（Duolingo）

Duolingo 2015 年论文《Half-Life Regression》。和 FSRS 思路类似但模型不同：

```
p(recall) = 2^(-Δ/h)

Δ = 距上次练习的时间
h = 记忆半衰期（类似 FSRS 的 S，但语义是"忘记一半所需时间"）
```

h 由回归模型预测：`h = exp(特征·权重)`，特征包括：练习次数、上次正确性、词难度、用户历史准确率等。

**与 FSRS 的区别**：
- FSRS 的 S 是**每卡每状态**的稳定性，Half-Life 的 h 是**特征驱动的回归**
- FSRS 模型结构固定（19 参数），Half-Life 可加任意特征（更灵活但更黑箱）
- FSRS 开源生态好（Anki/RemNote/MnemOS），Half-Life 是 Duolingo 内部用

**对你的含义**：如果你想加"特征"（如反应时、一天中的时间段、多模态输入类型）到调度，Half-Life 的思路可参考。但**不建议换掉 FSRS**——FSRS 已验证且你已集成。

### 1.8 Leitner System —— 纸牌时代的经典

5 个盒子，答对升一级（间隔变长），答错回第一盒。

```
盒子1: 每天
盒子2: 每3天
盒子3: 每周
盒子4: 每2周
盒子5: 每月
```

**现代意义**：Leitner 是离散版的 SM-2。一些简单 App（如 Mochi 的部分模式）仍用类似思路。你的"三阶递进"和 Leitner **神似**——都是离散跃迁——但跃迁依据不同（Leitner 是"答对/答错"，你的是"能力阶段"）。

### 1.9 MNEMOSYNE —— 数据收集的价值

MNEMOSYNE 是开源的 SM 变体（2006+），最大价值不是算法，而是**开放数据集**：积累了数百万条复习记录，用于学术研究。FSRS 的训练数据部分来自类似来源。

**对你的含义**：如果你能积累用户复习数据（特别是 L2/L3 的表现），未来可以训练自己的调度模型，或贡献给 FSRS 生态。

### 1.10 算法层对你架构选型的含义

| 决策点 | 建议 | 理由 |
|--------|------|------|
| 用 FSRS 还是 SM-2 | **保持 FSRS** | 已集成且更优，无理由回退 |
| desired_retention 可调 | **保留** | 备考冲刺期用户可调高，这是 FSRS 独有优势 |
| wordbook 级 weights | **保留** | 不同词库（基础词 vs 雅思）难度特征不同 |
| 加新信号维度 | **不进 FSRS，正交叠加** | L2/L3 表现、元认知偏差应是独立信号，不污染 D/S/R |
| 自研调度模型 | **暂不** | 数据量不够，等积累 10万+ 复习记录再考虑 |

## 第二部分：认知科学层 —— 理论到功能的转化

### 2.1 艾宾浩斯遗忘曲线（1885）

**原始实验**：Hermann Ebbinghaus 用无意义音节（如 "WID" "ZOF"）自我测试，发现遗忘的指数衰减规律。

**公式**：
```
R = e^(-t/S)

R = 记忆保留率（0-1）
t = 时间
S = 记忆强度（与个体、材料难度相关）
```

**重要细节常被误读**：
- 艾宾浩斯用的是**无意义音节**，不是真实学习材料。真实材料的遗忘曲线不同（有语义支撑的忘得慢）
- 曲线描述的是**再认/回忆的群体平均**，个体差异巨大
- "艾宾浩斯复习点"（1天/2天/4天/7天/15天）是**后人商业包装**，原文没有这些固定点

**到功能的转化**：
- ❌ 不要硬编码"艾宾浩斯复习点"——FSRS 的动态间隔已超越
- ✅ 可视化遗忘曲线（如你的 PlanVsActual）让用户"看到遗忘"——有教育意义
- ✅ retrievability 实时显示（你的 MasteryGlobe 已做）—— 让用户感受"这个词正在被忘记"

### 2.2 间隔效应（Spacing Effect）

**核心发现**：相同学习时间下，**分散学习**（spaced）比**集中学习**（massed/cramming）的长期记忆好得多。这是认知科学最稳健的发现之一（100+ 研究复现）。

**关键细节**：
- 间隔太短 ≈ 集中（无益）
- 间隔太长 ≈ 重新学（也无效）
- 最优间隔 ≈ 当前能记住时间的 10-20%（这就是 FSRS 在做的）
- 间隔效应在**再认**任务上效应小，在**回忆**任务上效应大——所以 cued recall / free recall 比 recognition 更受益于间隔

**到功能的转化**：
- ✅ FSRS 已在优化间隔——保持
- ✅ 多种提取类型（L2 cued / L3 free recall）比纯 recognition 更受益于间隔——这是"三阶递进"的理论支撑
- ❌ 不要提供"考前突击模式"（集中练习）——除非用户明确要短期记忆

### 2.3 测试效应（Testing Effect / Retrieval Practice）

**经典研究**：Roediger & Karpicke (2006)《Test-Enhanced Learning》。

**实验**：三组学生学同一篇文章：
- A 组：看 4 次
- B 组：看 1 次 + 测 3 次
- C 组：看 1 次 + 测 1 次 + 再看 2 次

**结果**（2 天后测试）：
- A 组（纯重读）：40%
- C 组（读+测+重读）：55%
- B 组（读+多测）：**60%**

**关键洞察**：测试（提取练习）比重读更有效，即使总时间相同。且**测试本身是学习**，不只是评估。

**提取类型的效果排序**（从强到弱）：
1. Free recall（自由提取，无线索）—— 最强
2. Cued recall（线索提取，如填空）—— 强
3. Recognition（再认，如选择题）—— 弱（但有，好过不测）
4. Re-reading（重读）—— 几乎无长期效果

**到功能的转化**：
- ✅ 每次复习必须有"提取"动作，不能只是"看卡片"——你的 zen 翻转是 recognition，效果最弱
- ✅ L2 的 cued recall（填空/辨析）比 L1 的翻转强
- ✅ L3 的 free recall / 造句是最强提取
- ❌ drill 的"看 cloze 选词"如果太容易就退化成 recognition——要确保有"提取困难"

### 2.4 Desirable Difficulty（Bjork 1994）

**核心概念**：Robert Bjork 提出，学习时的**适度困难**能增强长期记忆。太简单 → 编码浅；太难 → 挫败放弃；中等难 → 编码深。

**关键洞察**：
- "流畅性错觉"（illusion of fluency）：重读时觉得"我都会了"，实际没记住。流畅 ≠ 掌握
- 困难要"desirable"（合意的）：是认知负荷，不是挫败感。如：间隔、交错、测试、变换语境
- 困难会**降低即时表现但提升长期记忆**——这是反直觉的

**到功能的转化**：
- ✅ FSRS 的 desired_retention 调低（0.85）= 引入合意困难（让 R 到期时不是 100%）
- ✅ L2/L3 任务天然比 L1 难，是 desirable difficulty 的阶梯
- ✅ 交错练习（下条）是 desirable difficulty 的重要来源
- ❌ 不要让 UI 太"丝滑"——适度的"我想不起来"的痛苦是必要的。Zen 卡片翻转太快可能削弱提取努力

### 2.5 认知负荷理论（Sweller）

**三种负荷**：
| 类型 | 来源 | 设计目标 |
|------|------|---------|
| **内在负荷**（Intrinsic） | 材料本身难度 | 不可消除，分段可降低 |
| **外在负荷**（Extraneous） | 呈现方式的不良设计 | **应最小化** |
| **相关负荷**（Germane） | 用于建构图式的努力 | **应最大化** |

**到功能的转化**：
- ✅ 卡片设计要**减少外在负荷**：信息层级清晰、不堆砌、视觉聚焦（你的 ZenFlashcard 折叠 section 做得好）
- ✅ L1 的"分层渐进显示"是在控制内在负荷（不一次 dump 全部信息）
- ✅ L2/L3 任务增加相关负荷（用户要思考，建构图式）
- ❌ 卡片上放太多无关装饰 = 增加外在负荷，有害
- ❌ "花哨动画"如果分散注意力 = 外在负荷，有害

### 2.6 交错练习（Interleaving）vs 块状练习（Blocking）

**经典研究**：Rohrer & Taylor (2007)。学数学的学生：
- 块状组：先做 10 道题型 A，再做 10 道题型 B...
- 交错组：A/B/C/D 随机混合

**结果**：
- 即时测试：块状组好（块状有"假性掌握"）
- 一周后测试：**交错组显著好**

**关键洞察**：交错迫使大脑每次都"重新判断该用什么策略"，强化了**策略选择**而非"机械套用"。

**到功能的转化**：
- ✅ L2 随机抽题型（填空/辨析/词转）= 交错练习
- ✅ 复习队列混合不同语义场的词（不按 semantic_field 排序）= 交错
- ❌ 当前 drill 的 cloze/definition 二选一还不够交错——每次只一种
- ❌ "按词根分组复习"（如一次复习所有 ab- 开头的词）是块状练习，短期爽但长期差

### 2.7 元认知校准（Metacognitive Calibration）

**核心概念**：人对自己"知道多少"的判断（元认知）往往**不准**，且**系统性偏向过度自信**。

**经典研究**：Dunlosky & Rawson 等人。学习者预测自己能记住的比例，对比实际回忆率：
- 刚学完：预测 70%，实际 40%（过度自信 30%）
- 学了一段时间后：校准略好，但仍偏乐观

**影响校准的因素**：
- 再认 > 回忆：人们高估"能再认"为"能回忆"（你的 L1 recognition → L3 free recall 跃迁会暴露这个）
- 流畅性错觉：重读时觉得熟悉 = 以为记住了
- "差点想起"（TOT, tip-of-the-tongue）：是元认知信号，TOT 频繁 = 该词是 leech

**到功能的转化**：
- ✅ 方向 M（预测验证）的核心理论支撑
- ✅ L3 自评 vs L2 客观表现的差值 = 元认知偏差，可视化（校准曲线）
- ✅ "差点想起"标记（方向 F）= leech 早期信号
- ❌ 纯再认（zen 翻转）无法校准元认知——用户不知道自己"其实不会用"

### 2.8 上下文依赖记忆与状态依赖记忆

**经典研究**：Godden & Baddeley (1975)。潜水员在**水下/陆地**学单词，在**匹配环境**测试时回忆率高 50%。

**含义**：记忆与**学习时的上下文**绑定。变换上下文学习 → 在更多上下文能提取（编码特异性，Tulving）。

**到功能的转化**：
- ✅ 同一个词在不同语境出现（L2 的不同 corpus、L3 的造句）= 多上下文编码
- ✅ 不同复习模式（zen/drill/free/爬阶）= 不同认知状态，强化泛化
- ❌ 只在一种 UI 里复习 = 上下文单一，迁移到考试（不同环境）会打折

### 2.9 认知科学层总结：理论到功能映射表

| 理论 | 核心结论 | 对应功能方向 |
|------|---------|------------|
| 遗忘曲线 | 记忆指数衰减 | FSRS 调度（已做） |
| 间隔效应 | 分散 > 集中 | FSRS 间隔（已做） |
| 测试效应 | 提取 > 重读；free recall > cued > recognition | P 三阶递进（L2/L3 提取） |
| Desirable Difficulty | 适度困难增强长期记忆 | L2/L3 阶梯 + desired_retention 可调 |
| 认知负荷 | 减外在、增相关 | 卡片分层显示 + 任务驱动 |
| 交错练习 | 混合 > 块状 | L2 随机题型 + 队列混合语义场 |
| 元认知校准 | 人系统性过度自信 | M 预测验证 + L3 自评 |
| 上下文依赖 | 多上下文编码强化泛化 | 多模式 + 多语境语料 |
| 生成效应 | 自己产出 > 被动看 | F 自由提取 + L3 造句 |

## 第三部分：创新方向

### 3.1 LLM 个性化复习

**应用场景**：
- **造句评分**：用户造句，LLM 判语法/地道度/词用（方向 P 的 L3 原始设想，后改为自评）
- **个性化例句生成**：根据用户兴趣（从笔记/语料推断）生成例句，增强编码
- **弱点诊断**：LLM 分析用户的复习错误模式，给出"你总在 X 类词上失败"的诊断
- **对话式复习**：LLM 扮演对话伙伴，在对话中自然使用目标词

**风险与边界**：
- **延迟**：每次复习调 LLM = 300ms-2s 延迟，破坏复习节奏
- **成本**：高频复习场景的 API 成本不可忽视
- **一致性**：LLM 对相同输入可能给不同分，影响调度可靠性
- **离线**：LLM 不可用 = 功能不可用

**推荐策略**：
- LLM 用于**非调度路径**（不影响 FSRS 的辅助功能）：弱点诊断、个性化例句、对话练习
- **不用于**调度评分（用客观题或自评代替）
- 离线降级：LLM 不可用时跳过相关功能，不阻塞复习

### 3.2 多模态复习

**模态组合**：
| 模态 | 输入 | 输出 | 认知差异 |
|------|------|------|---------|
| 视觉（看词） | 文字 | 评级 | 当前 zen 模式 |
| 听觉（听写） | TTS 语音 | 拼写 | 拼写强化形-义映射 |
| 口语（跟读） | TTS 语音 | 录音+ASR | 发音+韵律 |
| 书写（手写） | 文字 | 手写识别 | 运动记忆强化 |

**认知科学支撑**：
- 多模态编码 = 多通道记忆痕迹，提取路径更多（双编码理论 dual coding, Paivio）
- 听写比再认难，属于 cued recall，对长期记忆贡献更大
- 手写（vs 打字）有运动记忆加成（James & Engelhardt 2014，手写激活更多脑区）

**到功能的转化**：
- 听写模式：TTS 读词 → 用户拼写 → 对比 lemma。可作为 L2 的一种任务
- 跟读模式：TTS 读例句 → 用户跟读 → ASR 评分发音
- 这些可作为**可选任务变体**，不强制

**你的项目已有基础**：lib/tts（TTS）、lib/haptics（触觉反馈），多模态有基础设施

### 3.3 群体智能难度数据

**思路**：收集所有用户的复习表现，聚合出"这个词对群体来说有多难"，用群体数据辅助个体调度（冷启动 + 校准）。

**实现方式**：
- 匿名上传复习记录（rating + 反应时 + 是否答对）
- 聚合出每词的群体难度（类似 Duolingo 的词难度）
- 新用户首次遇到该词时，用群体难度做 FSRS 初始 D 的先验

**风险**：
- 隐私（需明确同意 + 匿名化）
- 数据污染（恶意/噪声数据）
- 你是本地优先应用（LOCAL_OWNER 模式），群体数据基础设施不天然适配

**推荐**：暂不做。除非产品方向转为多用户 SaaS。

### 3.4 概念图谱驱动的复习（RemNote 路线）

**RemNote 的核心创新**：把笔记和闪卡合一。每个笔记节点可以"变成"卡片，卡片之间的关系 = 笔记的结构。

**对你的启发**：
- 你的词库已经是结构化的（六层内容），本质是一个轻量知识图谱
- 方向 R（神经通路）是这个思路的体现——复习关系而非孤立节点
- 更进一步：不只复习"词与词"的关系，还复习"词与语境""词与语义场"的关系

**RemNote 的局限**：图谱复杂后，用户维护成本高。你的优势是**词库是预制的**（Obsidian 笔记 + 导入脚本），用户不需要自己建图。

### 3.5 Desirable Difficulty 自适应

**思路**：系统动态调整难度，让用户始终处于"合意困难区"（不太难不太简单）。

**实现方式**：
- 追踪用户的"成功率"（最近 N 次答对率）
- 成功率 > 85% → 提升难度（升级到 L2/L3，或调高 desired_retention）
- 成功率 < 60% → 降低难度（回 L1，或调低 desired_retention）
- 这与 Bjork 的"desirable difficulty"直接对应

**到功能的转化**：
- 你的"爬阶会话达阈"就是一种粗粒度的自适应（good/easy 才升阶）
- 更细粒度：L2 任务的选项干扰度可调（答对率高 → 干扰项更接近；低 → 干扰项更明显）
- desired_retention 动态调整：连胜时调低（引入更多困难），连败时调高（给信心）

**风险**：过度自适应会让用户感觉"系统在操控"，需透明（告知为什么调难度）

### 3.6 Retrieval Practice 的多种变体

这是对 2.3 测试效应的展开——提取不止"翻转卡片"一种：

| 变体 | 描述 | 难度 | 你现状 |
|------|------|------|--------|
| Recognition | 看到选"认识/不认识" | 最低 | zen 四级评级偏此 |
| Cued recall (cloze) | 例句挖空填词 | 中 | drill 的 cloze |
| Cued recall (definition) | 看释义写词 | 中 | drill 的 definition |
| Cued recall (sound) | 听音写词 | 中高 | 无 |
| Free recall | 给主题列词 | 高 | 无（方向 F） |
| Serial recall | 按顺序列词 | 高 | 无 |
| Constructed response | 造句/解释 | 最高 | 无（L3） |
| Metacognitive judgment | 预测能否想起 | 元认知 | 无（方向 M） |

**到功能的转化**：
- 这些变体可以组合成"任务库"，系统按词的能力阶段 + 关系密度 + 用户偏好动态选
- 每种变体有独立的评分逻辑，但都映射到统一的能力阶段进度

### 3.7 创新方向对你项目的适用性评估

| 创新方向 | 适用性 | 理由 |
|---------|--------|------|
| LLM 个性化复习 | 中 | 用于辅助（诊断/例句），不用于调度 |
| 多模态复习 | 高 | 你已有 TTS/haptics 基础，听写/跟读是低成本扩展 |
| 群体智能难度 | 低 | 本地优先应用，不天然适配 |
| 概念图谱驱动 | 高 | 你的词库已是图谱，方向 R 是直接体现 |
| Desirable Difficulty 自适应 | 高 | 爬阶会话 + 动态 desired_retention 可实现 |
| Retrieval 变体库 | 高 | 任务库思路，与 P 三阶天然契合 |
