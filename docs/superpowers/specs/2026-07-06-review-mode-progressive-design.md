# 复习模式深度设计 —— 三阶递进复习

- **Status**: Draft / 探索阶段
- **Date**: 2026-07-06
- **Author**: brainstorming session
- **Context**: 基于 codebase-memory 架构验证 + 词库笔记架构调研 + 认知科学理论

## 背景

当前复习系统有三种模式（drill/free/zen），本质都是"卡片翻转 + 评级"范式的变体，交互高度同质。用户希望探索**复习模式的本质创新**——不一定是复习单词，而是设计"能力跃迁路径"，从"认识"到"能解题"到"能产出"。

### 现状基础（已验证）

- **算法**：FSRS v5/v6（ts-fsrs + native binding 优化器），19 个 w 参数，wordbook 级 weights，desired_retention=0.9
- **词库数据已就绪**：每个词条有六层结构化内容
  - 原型义（prototype_text）→ 词根叙事（morphology.parts + narrative）→ 语义链路（semantic_chain）→ 记忆锚点（mnemonic）→ 同义辨析（synonym_items 五维）→ 语料搭配（corpus_items + collocations）
- **关系图已有两套实现**：
  - 局部星型图 `buildLocalVocabGraph`（d3 force-directed，6 类边：root/synonym/antonym/prefix/suffix/related）
  - 全局掌握度网络 `MasteryNetworkCanvas`（three.js，retrievability 着色）
- **三种现有模式**：drill（自测操练无副作用）/ free（自由选词无 DB 写）/ zen（标准到期复习完整 FSRS）
- **痛点**：时区三套混用、v2 队列优先级退步、多 wordbook 不聚合、free 模式合成 session

## 三个候选方案

### 方案 A：单词内渐进（一次会话爬阶）—— 当前讨论主线

**核心**：每次复习一个词是"爬阶会话"——L1 锁定 → 达阈切 L2 辨析 → 再达阈切 L3 产出。一次会话走完能走的高阶。

**优势**：
- 最符合"递进"语义，产品辨识度高（没有别的 SRS 软件这么做）
- 一个词在一次会话里被多角度激活，记忆编码更深
- 失败不惩罚（标记薄弱，会话结束），节奏感强

**劣势**：
- 单词会话时间长（L1+L2+L3 走完可能 2-3 分钟/词）
- 状态机复杂，需要管理"当前阶段""达阈状态""薄弱标记"
- L3 产出任务可能打断复习心流

**适合场景**：用户愿意为"深度掌握"付出单词更长时间的复习

---

### 方案 B：三阶独立队列，用户切换

**核心**：L1/L2/L3 各自独立复习队列，用户今天主动选"我要练 L2 辨析"。阶段间有解锁门槛（L1 stability 达标才能解锁 L2），但不强制顺序。

**优势**：
- 最灵活，用户按当下状态选模式（碎片时间练 L1，大块时间练 L3）
- 每个队列内部逻辑简单（接近现有 zen 模式）
- 降级容易（不想练 L3 就不切）

**劣势**：
- "递进感"弱——用户可能永远只练 L1
- 三套队列三套调度，数据模型复杂
- 不符合"能力跃迁"的连续性直觉

**适合场景**：用户自律性强，能主动规划复习策略

---

### 方案 C：系统智能分配任务类型

**核心**：当天队列按词的能力阶段自动分配任务——L1 词走 L1 任务，达阈解锁的走 L2，再达阈走 L3。用户不选阶段，系统决定每个词今天做什么。

**优势**：
- 最省心，用户只管"开始复习"
- 队列天然混合（不会 20 个词全是 L1 单调）
- 复用现有 zen 的单卡流程，改造小

**劣势**：
- "递进感"最弱——用户感觉不到自己在"爬阶"
- 系统分配逻辑复杂（要兼顾阶段、FSRS 到期、题型多样性）
- 单次会话仍是"一卡一评"，没有爬阶体验

**适合场景**：用户想要"无脑复习"，系统兜底所有决策

## 方案对比

| 维度 | A 单词内渐进 | B 独立队列切换 | C 系统智能分配 |
|------|------------|--------------|--------------|
| 递进感 | ★★★ 最强 | ★☆☆ 弱 | ★★☆ 中 |
| 单词会话时长 | 长（2-3min） | 短（≈现有） | 短（≈现有） |
| 状态机复杂度 | 高 | 中 | 中 |
| 数据模型改动 | 中（加阶段字段） | 高（三套队列） | 中（加阶段字段） |
| 离线友好 | L3 产出可能受阻 | 各队列独立可离线 | 同现有 |
| 产品辨识度 | ★★★ 高 | ★☆☆ 低 | ★★☆ 中 |
| 改造量 | 大（新状态机+任务生成） | 大（三套队列） | 小（现有+任务分发） |

**推荐**：方案 A。理由是你的目标是"探索复习模式本质创新"而非"优化现有"，A 的辨识度和递进感最强，也最匹配"三阶能力跃迁"的设计意图。B 和 C 更像是"在现有范式上加题型"，创新性不足。

## 方案 A 详细设计（草稿）

### 核心模型：能力 ≠ 记忆强度

```
记忆强度（FSRS 管）：你还能不能想起这个词 → 横轴，随时间衰减，靠重复维持
能力阶段（新模式管）：你能把这个词用到什么程度 → 纵轴，靠"深度加工"跃迁，不随时间退回低阶
```

**关键决策**：三阶跃迁**不进 FSRS 调度**（避免污染记忆强度信号）。FSRS 仍管"什么时候复习"，三阶模式决定"复习时做什么"。

### 三阶定义

| 阶段 | 能力定义 | 提取类型 | 复习手段 | 用到的词库字段 |
|------|---------|---------|---------|-------------|
| **L1 认知** | 再认 + 发音 | Recognition | 标记 + 重复（发音/看卡片：词源+关系图+叙事） | lemma, ipa, short_definition, morphology, mnemonic, semantic_chain |
| **L2 应用** | 线索提取 + 辨析 | Cued recall | 语境填空、近义辨析、词性转换 | core_definitions, synonym_items, antonym_items, collocations, pos_conversions |
| **L3 产出** | 构造性产出 | Constructed response | 造句、语境重构、自由提取 | corpus_items, collocations.examples, semantic_chain |

### 爬阶会话状态机

```
进入会话（一个词）
    │
    ▼
┌─────────────────────────────────┐
│  L1 锚定阶段                     │
│  正面：lemma + ipa + 朗读        │
│  翻转：释义→原型义→词根叙事+     │
│       迷你关系图→语义链路        │
│  评级：again/hard/good/easy      │
│  达阈：good 或 easy → 进 L2      │
│  未达阈：会话结束，FSRS 照常记   │
└──────────┬──────────────────────┘
           │ good/easy
           ▼
┌─────────────────────────────────┐
│  L2 辨析阶段                     │
│  随机抽 1 种任务：               │
│  · 语境填空（corpus 挖空+4选1）  │
│  · 近义辨析（五维判断）          │
│  · 词性转换（pos_conversion）    │
│  评分：对/错 + 反应时 + LLM辅助  │
│  对 → 进 L3                      │
│  错 → 标记 L2 薄弱，会话结束     │
└──────────┬──────────────────────┘
           │ 答对
           ▼
┌─────────────────────────────────┐
│  L3 产出阶段                     │
│  随机抽 1 种任务：               │
│  · 造句（自评 + 对比真题语料）   │
│  · 语境重构（填词 + 对比原文）   │
│  评分：用户自评（不靠 LLM）      │
│  自评通过 → 能力跃迁到 L3        │
│  自评不通过 → 标记 L3 待练       │
└──────────┬──────────────────────┘
           │
           ▼
    会话结束，记录：
    · FSRS 评级（L1 的，进调度）
    · 能力阶段最高达到：L1/L2/L3
    · 薄弱标记：L2薄弱 / L3待练
    · 自评分数（如有）
```

### 失败处理：模式间分工

- **zen 模式**（爬阶会话）：L2/L3 失败不退回，标记薄弱，会话结束。失败是信号不是惩罚。
- **free 模式**（自由复习）：用户按"能力阶段""薄弱标记"筛选，针对性补练。安全网。

### L3 的特殊设计：自我校准（不依赖 LLM）

L3 的价值不是"系统判你对错"，而是**逼你面对真实语料，自己感受差距**：

1. **造句任务**：给目标词 + 语境提示，你造句
2. 造完不评分，展示**真题语料的真实用法**（corpus_items）让你对比
3. 你**自评**：地道度 / 词用准确性 / 语法，各 1-5 分
4. 系统记录自评，长期追踪**校准曲线**：自评 vs 下次 L2 客观表现

**为什么不用 LLM**：
- 用户终极目标是考试，最终裁判是考试不是机器
- 零 LLM 依赖，离线可用
- 训练元认知（备考者最需要的能力）
- 数据可追踪（自评 vs 客观表现差值 = 元认知偏差，是调度信号）

### 跃迁机制

- **升阶触发**：L1 的 FSRS stability 达阈值（如 ≥30 天）且最近 N 次 good/easy → 自动解锁 L2 任务
- **降阶保护**：不因遗忘退回低阶（能力是跃迁的），但 L2/L3 表现持续差会在 L1 队列里加权（更频繁复习基础）
- **手动跨阶**：用户可主动选"练 L3"，系统提示"该词 L1 稳定度还低，建议先巩固"

### 数据模型（待定，三种选项）

| 选项 | 描述 | 优劣 |
|------|------|------|
| **加列到 user_word_progress** | mastery_stage(int)、l2_weak(boolean)、l3_pending(boolean)、self_assessments(jsonb) | 改动小，复用现有表，但表会变宽 |
| **新建 user_word_mastery 表** | 与 user_word_progress 一对一 | 隔离关注点，但多表 + join |
| **不落库，纯前端状态** | 每次重新爬阶 | 最简单，但无法追踪长期进度 |

## 认知科学理论支撑

| 理论 | 在本设计的体现 |
|------|-------------|
| **测试效应**（Roediger & Karpicke 2006） | L2/L3 都是提取练习，比"重读"更有效 |
| **深度加工**（Craik & Lockhart） | L1→L2→L3 加工深度递增，从语义到构造 |
| **Desirable Difficulty**（Bjork 1994） | 爬阶本身就是"合意困难"，L2/L3 提供适度挑战 |
| **元认知校准**（Dunlosky） | L3 自评 vs 客观表现的长期对比 |
| **编码特异性**（Tulving） | L1 的关系图 + L2 的语境 + L3 的语料，多语境编码 |
| **生成效应**（Slamecka & Graf） | L3 造句是自己产出，比被动看记得牢 |
| **交错练习**（Bjork） | L2 随机抽题型，避免块状练习的假性掌握 |

## 算法层参考（不依赖联网，基于已有知识）

| 算法 | 与本设计的关系 |
|------|-------------|
| **FSRS（DSR 模型）** | L1 的调度核心，已采用。管"记忆强度"，不管"能力阶段" |
| **SM-2（Anki 基础）** | FSRS 的前身，本设计不回退到 SM-2 |
| **Half-Life Regression（Duolingo）** | 另一种记忆衰减模型，本设计不采用，但"半衰期"概念可用于解释 L1 stability |
| **Leitner System** | 5 盒子纸牌系统，本设计的"能力阶段"与之神似（都是离散跃迁），但跃迁依据不同 |
